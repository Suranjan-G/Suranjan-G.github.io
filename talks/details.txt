# Talks / Judging — content source
#
# HOW TO USE:
#   - Each entry is a block between "---" lines. Order in this file = order
#     on the rendered page (top = first / most prominent card).
#   - Add or remove entries freely. Blank fields are fine — they get skipped.
#   - After editing, run `python3 sync.py` from the site root to rebuild
#     /talks/index.html (and every other page). No dependencies.
#
# FIELDS:
#   type         "talk" or "judge"  — controls the small badge on the card
#   title        Talk title, or role for judging (e.g. "Hackathon Judge")
#   venue        Event / organiser + city, country
#   date         YYYY-MM-DD, YYYY-MM, or any short string (shown verbatim)
#   description  1–3 line description, plain text (can span multiple lines
#                as long as they're indented — see examples below)
#   media        Comma-separated filenames. All files go in
#                /talks/images+videos/. Mix images and video clips freely.
#                Extension controls the render: .jpg/.png/.webp → <img>,
#                .mp4/.webm/.mov → <video>.
#                Example: talk-nasiko-1.jpg, talk-nasiko-2.jpg, talk-nasiko-clip.mp4
#                One filename = single item (no carousel controls).
#                Two or more = auto-carousel with dots + counter + arrows.
#   slides       URL to slides deck, or leave blank
#   video        URL to external recording (e.g. YouTube), or leave blank
#   event        URL to event page, or leave blank
